避免內部邏輯曝露到外層

result = httpHandler.GetResponse()
if( result.returnCode == 200 ){
    // handler success
}

若 returnCode 要改成 200, 203 都handle success

那相關的code要改的地方就會很多

將判斷邏輯封裝，就可以避免這個問題

result = httpHandler.GetResponse()
if( result.IsSuccess ){
    // handler success
}

參數化

避免過於依賴從字典提取變數

  1. 會使得IDE無法發揮追蹤reference的功能
  2. 若很常從字典提取某個值，先拿到變數中，再繼續使用

    • Base

      a = aaa['a'] + 1
b = aaa['b'] + 5
c = aaa['c'] + 10

    • Better

      var dicA = aaa['a']
var dicB = aaa['b']
var dicC = aaa['c']

a = dicA + 1
b = dicB + 5
c = dicC + 10

善用 switch + enum，避免大量的if else

好處:

  1. 將情形明確條列
  2. 有default提醒預設行為
  3. 在看case的時候也有enum可以看到全部的情況，比一堆if else if好
  4. 可讀性比較高
  5. 擴充性比較好

  • Base

    if( httpStatus == 200 ){
    // do something
}else if( httpStatus == 401 ){
    // do something
}else{
    // do something
}

  • Better

    定義 enum

    public class HttpHandler{
    public enum StatusCode{
        Success = 200,
        Unauthorize = 401,
    }
}

    改寫 Base

    switch(httpStatus){
    case HttpHandler.StatusCode.Success:
        // do something
        break;
    case HttpHandler.StatusCode.Unauthorize:
        // do something
        break;
    default:
        // do something
        break;
}

Function向下兼容的更新方式

  1. 邏輯上更改: 要嚴謹的更改，讓既有之input output不受影響，但又能應對新邏輯，可以多靠測試來降低改錯的機會
  2. 擴充:

    1. 增加參數

      新增加之參數帶好default值，就不會影響既有code

      • Base:

        public void main(){
    Console.WriteLine(Add(1.3, 1.4)); // 2
}

public int Add(double a, double b){
    return (int)(a + b);
}

      • 擴充Add，以支援四捨五入

        public void main(){
    Console.WriteLine(Add(1.3, 1.4)); // 2
    Console.WriteLine(Add(1.3, 1.4, true)); // 2
}

public int Add(double a, double b, bool isRound = false){
    return isRound ? (int)(a + b) : (int)Math.Round(a + b);
}

    2. 多載method

      • 承上例之Base

        public void main(){
    Console.WriteLine(Add(1.3, 1.4)); // 2
    Console.WriteLine(Add(1.3, 1.4, true)); // 2
}

public int Add(double a, double b){
    return (int)(a + b);
}

public int Add(double a, double b, bool isRound){
    return isRound ? Add(a,b) : (int)Math.Round(a + b);
}

造成Bug的可能原因

  • 程式執行邏輯有誤
    • run time error
      • 程式沒寫好
        • 無法從物件中取值
        • 初始值有問題
      • 對語法不熟
        • static沒用好
    • 介接來源有問題
    • 商業邏輯有誤
      • 提出商業邏輯的人
        • 一開始提錯
        • 沒講清楚
      • 實作的人對規格錯誤認知
        • 誤認規格
        • 心理想的是對的，實作出來是不一樣的
  • 機器硬體有問題
    • 網路
    • 硬碟
    • 記憶體大小
    • CPU
