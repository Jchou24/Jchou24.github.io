由於網路上很多部署 .net core SPA 到 Heroku 的文章，因此

這邊不會贅述詳細流程，也不會分析使用 Heroku 的優劣

這篇文章主要想記錄的，是我部署的過程中遇到的一些小細節

部署流程概述

  1. 到 Heroku 上註冊帳號 (如果沒有帳號)

  2. 安裝 Heroku cli (如果沒有)

  3. 到 Heroku 上建立新的app

  4. 設定 buildpack

  5. 到 Heroku 上建立新的 Posgres DB

  6. 預先將 SPA 的部分 build 好並遷入 git

  7. 直接 git push heroku HEAD:master 觸發網頁部署

  8. 指定 Heroku app type

設定 buildpack

首先，Heroku好像是設計機制的關西吧，他有幾種部署模式

那如果要使用 git push heroku HEAD:master 的方式部署

目前看來，似乎必定會觸發編譯的過程

也就是說，他不允許在這種部署模式下讓你直接上傳編譯好的檔案

而 .net core 預設不在官方支援的編譯類型當中

因此要使用到一位大大寫的編譯指令，dotnetcore-buildpack

其實看一下他寫的指令，基本上就是一串 shell script，直接下載 .net core 在 linux 上的cli

然後用那個 cli 進行編譯，並寫好網站執行的指令等等等(裡面其實也是做了不少事)

而安裝buildpack，可以透過以下指令進行

heroku buildpacks:add https://github.com/jincod/dotnetcore-buildpack

或直接到 heroku 的 dashboard 上，用 GUI 介面設定

那這邊還有一個issue，就是SPA編譯的部分

由於目前 Heroku 官方寫的編譯 buildpack(heroku/nodejs) 要求 package.json 必須在根目錄上不能在子目錄

所以...，我看了一下相關的文章是說，這好像跟 Heroku 想要 cache node_modules 有關

但...簡單來說，要不放子目錄，不然就不要用 Heroku

或者，就暴力解吧，直接預先編譯好SPA的部分後遷入 git

我有嘗試想要使用子目錄編譯的東西，像是 https://github.com/timanovsky/subdir-heroku-buildpack

但這套件居然會在編譯好之後把其他東西清掉...Orz

還有的作法是說，用 git subtree 去指定路徑

好吧，對於我這個只是想上傳 demo 網站的人來說，我決定還是直接遷入好了 lol

到 Heroku 上建立新的 Posgres DB

再來是建立DB的部分，基本上就是把以下的欄位照著填，然後記得補上SSL相關的設定

如果照著填還發生DB連不到的問題，有很大機率不是連線字串的問題

可能是其他的設定，或者是套件版本的問題

要填的相關欄位有:

欄位 用途
User ID DB 帳號
Password DB 密碼
Server DB 的位置
Port DB Port號(option)，預設為5432
Database DB 名稱
Pooling 連線機制(option)
SSL Mode 連線機制
Trust Server Certificate 連線機制

最終組出來的連線字串會像這樣

User ID=<YOUR_ID>;Password=<YOUR_PWD>;Server=<YOUR_SERVER_IP>;Port=5432;Database=<YOUR_DB_NAME>;Pooling=true;SSL Mode=Require;Trust Server Certificate=true

設定連線字串

如果不希望DB的一些機敏資料遷入 git 中，也可以選擇讓程式去讀 Environment Variable

把連線字串寫在 env var 中，這樣就可以避免這個問題

直接 git push heroku HEAD:master 觸發網頁部署

首先要加入一個 remote

heroku git:remote -a <YOUR_HEROKU_APP_NAME>

再來，就是 push 到 heroku 上

簡稱指令是 git push heroku master

如果不行的話，就要用完整一點的指令 git push heroku HEAD:master

或者就要看你怎麼規畫你的 git 分支

指定 Heroku app type

如果啟動網站後看不到網站，而是 error page

看 log 還有發現以下的字樣的話

at=error code=H14 desc=”No web processes running” method=GET path=”/” host=xxx.herokuapp.com request_id=xxx fwd=”xxx.xx.xx.x" dyno= connect= service= status=503 bytes=

那8成就是 Heroku 誤判了這個 Heroku app 的類型

必須要手動下指令讓他知道這是一個網站

heroku ps:scale web=1

觀看設定檔

當部署完成後，如果想調整 appSettings.json 或一些設定檔，可以登入上去調

heroku login # 先進行登入認證
heroku run bash # 這樣就能進 server 用 linux 指令去進行操作了
